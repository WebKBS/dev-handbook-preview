설치부터 배포까지: 한 번에 이해하기

기술 문서는 설명과 코드가 섞여 있어요. 본문은 넉넉하게(1.7), 코드는 조금 타이트하게(1.5) 잡으면 줄 간격 차이가 자연스럽게 드러납니다.

이 문서에서 다루는 내용

  1. 프로젝트 설치
  2. 환경 변수 설정
  3. 로컬 실행
  4. 빌드 및 배포

1) 프로젝트 설치

패키지 매니저는 무엇이든 가능하지만, 예시는 pnpm 기준입니다. 아래 명령으로 의존성을 설치하세요. 

# install dependencies
pnpm i

# run dev server
pnpm dev

2) 환경 변수 설정

로컬 개발에서는 .env 파일을 사용합니다. API 주소를 VITE_API_BASE_URL로 관리한다면 다음처럼 작성할 수 있어요. 

# .env
VITE_API_BASE_URL=https://api.example.com
VITE_APP_ENV=local

# optional
VITE_ENABLE_MOCK=true

3) 간단한 요청 예시

아래는 “동작 확인” 용도로 한 번쯤 넣는 예시입니다. 코드블럭은 줄 간격이 너무 넓으면 헐렁해 보여서 line-height: 1.5가 보통 더 깔끔합니다. 

curl -X GET "https://api.example.com/health" \
  -H "Accept: application/json"

# response example
# { "ok": true }

4) 요약

본문은 line-height: 1.7로 여유를 주고, 코드/명령은 line-height: 1.5로 타이트하게 잡으면 “설명”과 “코드”의 리듬이 분리돼 읽기 편해집니다.

요약

  • 본문: 16px / line-height 1.7 (설명 가독성 우선)
  • 코드: line-height 1.5 (라인 간격 과확장 방지)
